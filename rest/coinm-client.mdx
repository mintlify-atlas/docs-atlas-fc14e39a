`CoinMClient` is the dedicated REST client for Binance's COIN-M Futures market. Unlike USD-M Futures, COIN-M contracts are **settled and margined in the underlying cryptocurrency** â€” for example, a `BTCUSD_PERP` position uses BTC as collateral and is settled in BTC. The client targets the `dapi*.binance.com` subdomains and provides typed methods for market data, order management, position control, and account operations across all COIN-M Futures instruments.

<Warning>
  COIN-M Futures use the underlying asset (e.g. BTC, ETH) as both margin and settlement currency. Profits, losses, and fees are denominated in that asset. Ensure you understand coin-denominated P&L before trading.
</Warning>

## Installation

<Steps>
  <Step title="Install the package">
    ```bash
    npm install binance
    ```
  </Step>
  <Step title="Import CoinMClient">
    ```typescript
    import { CoinMClient } from 'binance';
    // CommonJS: const { CoinMClient } = require('binance');
    ```
  </Step>
  <Step title="Instantiate the client">
    ```typescript
    const client = new CoinMClient({
      api_key: process.env.BINANCE_API_KEY,
      api_secret: process.env.BINANCE_API_SECRET,
    });
    ```
  </Step>
</Steps>

## Constructor Options

```typescript
const client = new CoinMClient(
  restClientOptions,  // RestClientOptions (optional)
  requestOptions,     // AxiosRequestConfig (optional)
);
```

<ParamField body="api_key" type="string">
  Your Binance API key. Required for all authenticated (private) endpoints.
</ParamField>

<ParamField body="api_secret" type="string">
  Your Binance API secret. Supports HMAC, RSA, and Ed25519 â€” key type is detected automatically.
</ParamField>

<ParamField body="beautifyResponses" type="boolean" default="false">
  When `true`, numeric strings in responses are parsed into JavaScript numbers for well-known fields.
</ParamField>

<ParamField body="testnet" type="boolean" default="false">
  Routes requests to `https://testnet.binancefuture.com` (COIN-M testnet). Uses simulated market data.
</ParamField>

## COIN-M vs USD-M Key Differences

<Tabs>
  <Tab title="COIN-M Futures">
    - **Base URL:** `dapi*.binance.com`
    - **Settlement currency:** Underlying asset (BTC, ETH, BNB, etc.)
    - **Symbol format:** `BTCUSD_PERP`, `ETHUSD_PERP`, `BTCUSD_240329` (delivery)
    - **Margin:** Denominated in the underlying coin
    - **P&L:** Expressed in the underlying coin
    - **Use case:** Naturally long the underlying coin; ideal for miners and holders who want leveraged exposure without selling their crypto
  </Tab>
  <Tab title="USD-M Futures">
    - **Base URL:** `fapi*.binance.com`
    - **Settlement currency:** USDT or BUSD
    - **Symbol format:** `BTCUSDT`, `ETHUSDT`
    - **Margin:** Denominated in USDT
    - **P&L:** Expressed in USDT
    - **Use case:** Stable-coin denominated P&L; simpler for accounting in fiat terms
  </Tab>
</Tabs>

## Code Examples

<CodeGroup>
```typescript Public Market Data
import { CoinMClient } from 'binance';

const client = new CoinMClient();

async function fetchCoinMData() {
  // Exchange info â€” note BTCUSD_PERP symbol format
  const info = await client.getExchangeInfo();
  const perps = info.symbols.filter((s) => s.contractType === 'PERPETUAL');
  console.log(`${perps.length} perpetual contracts available`);

  // Mark price and funding rate for BTC perpetual
  const markPrice = await client.getMarkPrice({ symbol: 'BTCUSD_PERP' });
  console.log('Mark price:', markPrice.markPrice);
  console.log('Funding rate:', markPrice.lastFundingRate);

  // Order book
  const book = await client.getOrderBook({ symbol: 'BTCUSD_PERP', limit: 5 });
  console.log('Best bid:', book.bids[0]);
  console.log('Best ask:', book.asks[0]);
}

fetchCoinMData().catch(console.error);
```

```typescript Authenticated Trading
import { CoinMClient } from 'binance';

const client = new CoinMClient({
  api_key: process.env.BINANCE_API_KEY!,
  api_secret: process.env.BINANCE_API_SECRET!,
  beautifyResponses: true,
  // testnet: true, // switch to testnet for testing
});

async function tradeCoinM() {
  // Check balance â€” returned in BTC, not USDT
  const balances = await client.getBalance();
  const btcBalance = balances.find((b) => b.asset === 'BTC');
  console.log('BTC balance:', btcBalance?.balance);

  // Set 5x leverage on BTC perpetual
  await client.setLeverage({ symbol: 'BTCUSD_PERP', leverage: 5 });

  // Place a market buy (1 contract = 100 USD notional for BTC)
  const order = await client.submitNewOrder({
    symbol: 'BTCUSD_PERP',
    side: 'BUY',
    type: 'MARKET',
    quantity: 1,
  });
  console.log('Order:', order.orderId, order.status);

  // Query position â€” P&L is in BTC
  const positions = await client.getPositionInformation({ symbol: 'BTCUSD_PERP' });
  console.log('Position:', positions[0]?.positionAmt, 'BTC notional');
}

tradeCoinM().catch(console.error);
```

```typescript Order Book Ticker
import { CoinMClient } from 'binance';

const client = new CoinMClient();

async function checkTicker() {
  // All COIN-M order book tickers
  const tickers = await client.getSymbolOrderBookTicker();
  console.log(`${tickers.length} symbols`);

  // Single symbol
  const btcTicker = await client.getSymbolOrderBookTicker({
    symbol: 'BTCUSD_PERP',
  });
  console.log('BTC best bid:', btcTicker[0]?.bidPrice);
  console.log('BTC best ask:', btcTicker[0]?.askPrice);
}

checkTicker().catch(console.error);
```
</CodeGroup>

## COIN-M Symbol Naming

COIN-M symbols follow a different convention from Spot and USD-M Futures:

| Format | Example | Description |
|--------|---------|-------------|
| `{BASE}USD_PERP` | `BTCUSD_PERP` | Perpetual contract |
| `{BASE}USD_{YYMMDD}` | `BTCUSD_240927` | Quarterly delivery contract |
| `{BASE}USD_{YYMMDD}` | `BTCUSD_241227` | Next-quarter delivery contract |

<Info>
  The `pair` parameter (e.g. `BTCUSD`) can be used in some COIN-M endpoints instead of `symbol` to retrieve data for all contracts on a given underlying asset simultaneously.
</Info>

## Testnet Support

COIN-M Futures testnet is available at `https://testnet.binancefuture.com`. Pass `testnet: true` in the constructor to route all requests there. Note that COIN-M Demo Trading is not currently available â€” the testnet is the only sandbox environment for COIN-M.

```typescript
const client = new CoinMClient({
  api_key: 'YOUR_TESTNET_KEY',
  api_secret: 'YOUR_TESTNET_SECRET',
  testnet: true,
});
```
